### 一、 实验目的

1. 深入理解现代 Transformer 架构的核心基石——缩放点积注意力（Scaled Dot-Product Attention）的底层数学机制。

2. 掌握并严格验证注意力计算中的高维张量变换：包括 Batch 维度下的矩阵相乘（`torch.matmul`）、维度的动态匹配，以及 Softmax 归一化。

3. 深刻理解**位置编码（Positional Encoding）**的不可或缺性：由于注意力机制天生具备排列不变性，必须引入位置特征才能让模型感知序列的顺序坐标。

4. 构建并训练一个基于交叉注意力（Cross-Attention）的合成任务“指针检索模型”，直观观察 Attention 权重分布，体会模型的高可解释性。

### 二、 实验数据与网络架构

- **数据集：** 构造的一组合成指针检索数据（Pointer Retrieval）。词表大小 $Vocab=20$，序列长度固定为 $L=16$。每个样本由随机生成的长度为 $L$ 的序列 $x$、一个代表检索目标的指针坐标 $p \in [0, L-1]$ 以及真实的检索结果 $y = x[p]$ 组成。

- **缩放点积注意力算子：**
  
  - 核心公式：$Attention(Q, K, V) = \text{softmax}(\frac{Q K^T}{\sqrt{d_k}})V$
  
  - 输入形状：$Q \in \mathbb{R}^{B \times 1 \times d_k}, K \in \mathbb{R}^{B \times L \times d_k}, V \in \mathbb{R}^{B \times L \times d_v}$

- **指针注意力模型：**
  
  - **序列映射 (Key, Value)：** 序列通过词向量 `embedding(x)` 提取特征，**并与位置向量 `pos_embedding(positions)` 逐元素相加**，融合内容与位置坐标后，经线性层映射为 $K, V$。
  
  - **指针映射 (Query)：** 将标量坐标 $p$ 转换为 One-Hot 向量，经映射得到专属查询特征 $Q$。
  
  - 融合后输出送入全连接层进行 $Vocab$ 的多分类预测。

### 三、 实验步骤与核心算法

1. **手写 Attention 函数：** * 使用 `K.transpose(-2, -1)` 完成 Batched Matrix Multiplication (BMM)。
   
   - 添加缩放因子 $\sqrt{d_k}$ 防止点积过大进入 softmax 饱和区导致梯度消失。

2. **张量底层自检：** * 在 Attention 第一次调用时触发自检。日志成功验证 `Out shape: [64, 1, 32]` 以及 `Attn shape: [64, 1, 16]`，且 $attn.sum(dim=-1)$ 为 `1.0000`，概率归一化严格正确。

3. **修复“注意力盲区”：** * 实验初期发现模型 Loss 无法收敛且准确率极低。经排查，由于 Key 和 Value 缺乏位置信息，导致 Query 无法寻址。通过引入 `nn.Embedding(seq_len, d_model)` 作为位置编码并与词向量相加，成功激活了模型的定位能力。

4. **模型训练：**
   
   - 采用 `CrossEntropyLoss` 及 `Adam` (lr=1e-3) 进行优化。引入位置编码后，Loss 在数个 Epoch 内断崖式下跌，准确率极速攀升至近 100%。
